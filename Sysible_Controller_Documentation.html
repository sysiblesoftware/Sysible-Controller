• Read the follow-up dialog: if you're reusing this bundle on a host that has run an older Sysible agent before, delete any leftover agent_state.json on that host first, or the agent may try to resume an identity the controller no longer recognizes.

    • Tag a host with an environment

    1. In the Enrolled Hosts list (grouped by environment; use Collapse All / Expand All to navigate a large fleet), select a host.
    2. Check its Details box (Hostname, Host ID, Platform, Kernel, Status, Environment).
    3. In the Environment combo, pick an existing environment, or choose + New environment... and type a new name when prompted.
    4. Click Set Environment.

    Refresh the enrolled hosts list

    1. Click Refresh.

    Disenroll a host

    1. Select the host, click Disenroll Host.
    2. Confirm in the dialog that appears.
    3. The controller queues an uninstall task on that host's agent and waits (polling roughly every two seconds, for up to about twelve seconds) for it to acknowledge.
    4. If the host acknowledges, its agent service is removed and the host drops out of the list.
    5. If the host is offline or doesn't respond in time, it is removed from the controller's records anyway, with a note that you should run disenroll_agent.sh on that host yourself later if you also want its local agent service cleaned up.

    Remote Host Administration

    Dashboard tile: Remote Host Administration. See Section 8 for the terminal and file-transfer model.

    Connect to a new host over SSH

    1. In the SSH to a New Host (Not Yet Joined) section, fill in Host name, IP address, Username (defaults to root), and the host's current SSH password.
    2. Choose an Environment, or + New environment... to create one.
    3. Click Connect Host. The controller installs its own SSH key on the host using the password you gave it once, then discards that password — every connection afterward is passwordless.
    4. The host now appears in Managed Hosts.

    Open a terminal session on a managed host

    1. Select a host in Managed Hosts.
    2. SSH hosts open a live, fully interactive terminal — sudo prompts, vim, arrow keys, and Ctrl+C (via the dedicated Send Ctrl+C button, which only appears during a live SSH session) all work as they would in a normal terminal.
    3. Agent hosts use a type-and-submit model instead: type a command and send it, wait while it's queued and polled (up to 30 seconds), then read the result once it returns. There's no partial output and no way to interrupt a running command.
    4. Click Clear to clear the terminal pane's contents.

    Upload a file to a host

    1. Click Browse... and choose a local file.
    2. Enter the destination path on the remote host.
    3. Click Upload.
    4. SSH hosts transfer synchronously over SFTP with no size limit. Agent hosts queue the transfer through the same task channel as commands, capped at the agent file-transfer size limit, and it's polled until done.

    Download a file from a host

    1. Enter the source path on the remote host.
    2. Click Download... and choose where to save it locally.
    3. Same SSH-vs-agent behavior as uploading: immediate SFTP for SSH, capped and polled for agent hosts.

    Remove a host

    1. Select it in Managed Hosts, click Remove Host.
    2. Confirm. If the row represents a merged agent+SSH entry, both connections are removed in this one action; if only one side fails, that error is reported but the other side's removal still goes through.

    View the controller's SSH public key

    1. Click Show Sysible Controller Public Key.
    2. Copy it if you'd rather pre-authorize a host yourself — baked into a golden image, for instance — instead of using Connect Host's one-time password handoff.

    AppendixCLI Quick Reference

    A condensed reference for sysible_controller — see Section 5 for full detail.

    sysible_controller {start|stop|restart|status|logs|gui|destroy}
    CommandRoot?Summary
    startYesStarts the backend, confirms the API is actually reachable, then launches the GUI.
    stopYesStops the GUI, the backend, the portal, and anything still bound to port 9000.
    restartYesstop then start.
    statusNoBackend systemd status plus GUI process state.
    logsNoTails the backend's live log stream.
    guiYesStarts only the GUI — use to reopen the dashboard after it's been closed.
    destroyYesIrreversible uninstall. Requires typing destroy to confirm. Backs up the database to /tmp first; does not touch already-enrolled hosts.
