Most tools share the same pattern: open the tool, check the hosts to act on in the Target Hosts list, then click an action — each host's result appears in its own tab with a green (success) or red (failure) banner.

The Dashboard

  1. Log in; you land on the Dashboard — a search box over a column of tiles.
  2. Type a task like create a user or open a firewall port to jump straight to the right tool.
  3. Or click a tile to open that tool in its own window. (The Live Activity & Logs tile appears only for Superusers.)

Live Activity & Logs (Superusers only)

  1. Open Live Activity & Logs from the Dashboard — visible only to Superuser administrators.
  2. The Activity tab shows a live, plain-language feed of who did what across the fleet; the Controller Log tab tails the backend's own log.
  3. Both auto-refresh; routine background syncs are filtered out so the feed reflects real actions.

Host Enrollment

  1. Open Sysible Controller Host Enrollment from the Dashboard.
  2. Set a default environment, then click Download Agent Bundle and install it on the target host.
  3. Enrolled hosts appear in the list; shift/ctrl-click several at once to Set Environment in bulk, set each host's Sudo Mode (NOPASSWD or password-required), or Disenroll Host.

Sysible Connect

  1. Open Sysible Connect. Agent and SSH hosts share one list, each with its IP.
  2. Double-click a host to open a pop-out terminal (it opens as your admin user, not root); or click Open RDP… for a graphical desktop, and Send sudo password to answer a sudo prompt from your encrypted store.
  3. Use Fleet Power Controls (Reboot / Power Off / Restart Agent on all hosts) or Run Script on All Hosts… to act fleet-wide. New SSH host? Fill SSH to a New Host and Connect Host — the password is used once, then replaced by a key.

The Terminal

  1. Each terminal is a real PTY — vim, top, and less render correctly.
  2. The user@host prompt is green (red for root); the toolbar adds upload/download, find, save-output, and font sizing.
  3. Open as many terminals as you like — sessions are independent, even multiple to the same host.

Graphical Desktop (RDP)

  1. Select a host in Sysible Connect and click Open RDP….
  2. Enter host, username, password, and resolution (the default fits the window).
  3. Optionally remember credentials — they're encrypted on your workstation. Click Connect to launch FreeRDP (or Remmina).

User & Group Administration

  1. Open System Administration → User & Group Administration.
  2. Check one or more hosts in the Target Hosts list (use Select All for the whole fleet).
  3. Use the Create User / Account / Password / Groups tabs and click an action (e.g. Create User, Lock, Toggle Sudo). Each host's result opens in its own tab.

System Health, Logs & Recovery

  1. Open System Administration → System Health, Logs & Recovery (one window, two top-level tabs: Health & Logs and Boot & Recovery).
  2. Check one or more hosts in the Target Hosts list (use Select All for the whole fleet).
  3. On Health & Logs, click Check Host Health for a green/amber/red verdict, or use the Overview/Processes/Logs/Diagnostics tabs. On Boot & Recovery, analyze boot failures, manage GRUB and kernels — with care.

Service Management

  1. Open System Administration → Service Management.
  2. Check one or more hosts in the Target Hosts list (use Select All for the whole fleet).
  3. Type a service name and click Start, Stop, Restart, Enable At Boot, Status, or Logs; Troubleshoot diagnoses failures.

Environmental Policies

  1. Open System Administration → Environmental Policies.
  2. Check one or more hosts in the Target Hosts list (use Select All for the whole fleet).
  3. Set the password, lockout, sudo, and umask baseline, then click Push Selected Policies to Checked Hosts.

Cron & Systemd Timers

  1. Open System Administration → Cron & Systemd Timers.
  2. Check one or more hosts in the Target Hosts list (use Select All for the whole fleet).
  3. Use List Cron Jobs / Add Cron Job (the schedule builder writes the syntax for you), or manage timers with Create Timer, Start, Enable At Boot.

Host Software Management

  1. Open System Administration → Host Software Management.
  2. Check one or more hosts in the Target Hosts list (use Select All for the whole fleet).
  3. Click Detect Package Manager / OS, then Install, Remove, Update / Upgrade, or Query Package Info — the same buttons work across apt/dnf/yum/zypper.

Repository Management

  1. Open System Administration → Repository Management.
  2. Check one or more hosts in the Target Hosts list (use Select All for the whole fleet).
  3. Click List Repositories, then Add Repository (or Add Repository to All Hosts), Enable, Disable, or Remove Repository.

Network Management

  1. Open System Administration → Network Management.
  2. Check one or more hosts in the Target Hosts list (use Select All for the whole fleet).
  3. Start in Diagnostics (Ping, Traceroute, Test DNS Resolution); use the other tabs to configure addressing, DNS, routing, and bonds/VLANs/bridges.

File System Management

  1. Open System Administration → File System Management.
  2. Check one or more hosts in the Target Hosts list (use Select All for the whole fleet).
  3. Start in Directories and Files — List Directory shows a path's contents; then create/copy/move/rename. Separate tabs cover permissions/links, Mount / Unmount, Network Mounts (NFS/CIFS), Resize & Repair, Disk Usage, fstab/quotas, and archives.

Storage Administration

  1. Open System Administration → Storage Administration.
  2. Check one or more hosts in the Target Hosts list (use Select All for the whole fleet).
  3. Work through the tabs: Disks (and SMART), Partitions, Format, LVM, RAID, Swap. Enter a device like /dev/sdb and click the action.

Firewall Administration

  1. Open System Administration → Firewall Administration.
  2. Check one or more hosts in the Target Hosts list (use Select All for the whole fleet).
  3. Use Firewalld/Ports to Open Port / Close Port and Reload; Zones, Rich Rules, and raw nftables/iptables tabs are there for advanced rules.

Security Administration

  1. Open System Administration → Security Administration.
  2. Check one or more hosts in the Target Hosts list (use Select All for the whole fleet).
  3. Tabs: SELinux, SSH (harden + rotate keys), Audit & Logins, Updates & Policy, and Hardening & Scans (Lynis/rkhunter).

Backup & Recovery

  1. Open System Administration → Backup & Recovery.
  2. Check one or more hosts in the Target Hosts list (use Select All for the whole fleet).
  3. Set a source and destination and click Back Up, then Verify or Restore; the other tabs cover LVM snapshots and scheduled/DR drills.

Time Synchronization

  1. Open System Administration → Time Synchronization.
  2. Check one or more hosts in the Target Hosts list (use Select All for the whole fleet).
  3. Click Sync Status / Verify Synchronization, then Set NTP Servers or Set Time Zone as needed.

Certificate Management

  1. Open System Administration → Certificate Management.
  2. Check one or more hosts in the Target Hosts list (use Select All for the whole fleet).
  3. Click Generate CSR, Install, or Check Expiry; the Chain & TLS tab verifies chains and troubleshoots endpoints.

Containers & VMs

  1. Open System Administration → Containers & VMs.
  2. Check one or more hosts in the Target Hosts list (use Select All for the whole fleet).
  3. Click Detect Runtime, then List Containers, View Container Logs, Run, or Prune; the Virtual Machines tab manages libvirt VMs.

Directory Services (Active Directory / LDAP)

  1. Open System Administration → Directory Services and check the target hosts.
  2. Click Prepare Host for AD Join first — it installs the tooling, syncs time, enables home-dir creation, and verifies the domain's SRV records resolve.
  3. With a clean readiness report, click Join Domain then Permit Login; the LDAP / LDAPS and Kerberos tabs configure and test identity.

Distro Subscription & Licensing

  1. Open System Administration → Distro Subscription & Licensing.
  2. On Overview, click Detect Distro & Subscription Tools to see which vendor applies; check the hosts to act on.
  3. Use the matching tab — Red Hat (RHSM) (Register All registers + auto-attaches every checked host at once), Ubuntu Pro (attach a token, enable services), or SUSE (SCC) (register a reg-code) — then Status / List to confirm.

Sysible Controller Settings

  1. Open Sysible Controller Settings from the Dashboard.
  2. Set the controller address/port, and add administrators choosing a role — Superuser (manage admins, see live logs) or Sysadmin (fleet tools only). Tune the admin password policy.
  3. Review the Audit Log and record a license key under License & Version.

Webserver Portal

  1. Open Webserver Portal Configuration from the Dashboard.
  2. Set a port and credentials, then click Start Portal.
  3. Share the URL with host operators; manage the shared file pool and review login history/sessions here.
